# vDAQ README 改版设计

## 目标

将 README 重写为面向 GitHub 展示、项目复现和嵌入式面试的工程首页，准确反映
阶段 4 与模块化重构后的真实状态。README 保持紧凑，原理细节继续由 `docs/` 承载。

## 内容结构

1. 项目定位：说明虚拟多通道 DAQ 的目标、运行平台和学习价值。
2. 已实现能力：字符设备、hrtimer、Ring Buffer、ioctl、阻塞/非阻塞 I/O、
   poll/epoll LT/ET、并发保护和模块化结构。
3. 架构与数据流：展示 main、buffer、control 的职责及用户态到驱动的数据链路。
4. 目录结构：仅列源文件和关键文档，不展示构建产物。
5. 构建与运行：使用真实路径和 `app/build/` 命令，修正状态查询为 `get_status`。
6. 接口说明：概括采样格式、ioctl、read 和事件通知语义。
7. 验证结果：记录已经真实执行的构建、功能、epoll、STOP/HUP 与 50 次重载测试。
8. Roadmap：阶段 5 sysfs/debugfs、故障注入、恢复状态机、压力测试、mmap 和
   暂缓的 HC-SR04 真实硬件接入。
9. 深入文档：链接现有架构、驱动设计、ioctl、测试报告和开发日志。

## 准确性约束

- 不声称尚未实现的 sysfs、debugfs、故障恢复或 HC-SR04 已完成。
- 明确物理数组为 1024 槽，但当前空槽判定方案的有效容量是 1023 帧。
- 不虚构性能数据，只列已经运行过的验证。
- 不修改 UAPI、源码、测试脚本或超声波笔记。
- README 使用简体中文，保留必要的英文内核术语和命令。

## 验收

- README 中的路径、目标名和 CLI 命令均能在当前仓库对应。
- Markdown 代码块、表格和链接格式正确。
- 已完成与 Roadmap 分界明确。
- `git diff --check` 通过，且提交只包含 README 与本设计文档。
